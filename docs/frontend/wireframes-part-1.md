## 0. نظام الشبكة والنقاط الفاصلة (Grid & Breakpoints)

| الجهاز | العرض | عدد الأعمدة | الهوامش الجانبية | المسافة بين الأعمدة |
|---|---|---|---|---|
| **Mobile** | < 640px | 4 columns | 16px | 12px |
| **Tablet** | 640–1024px | 8 columns | 32px | 16px |
| **Desktop** | > 1024px | 12 columns | Max-width 1280px، مركزي | 24px |

**قواعد عامة عبر كل الصفحات:**
- ترويسة (Header) ثابتة (`sticky top-0`) في كل الأجهزة.
- Mobile: تنقل رئيسي عبر **Bottom Navigation** ثابت (`fixed bottom-0`) بأربعة عناصر: الرئيسية، المتجر، الأكاديمية، حسابي.
- كل صورة/فيديو له `aspect-ratio` محدد مسبقًا لمنع Layout Shift (مطلب أساسي في Core Web Vitals لعام 2026).
- استخدام Skeleton Screens (لا Spinner) في كل حالة تحميل بيانات، بنفس شكل المحتوى المتوقع.
- الاتجاه (RTL/LTR) ينعكس على كل الهيكل تلقائيًا — الأيقونات الاتجاهية (أسهم، Chevrons) تُعكس أيضًا.

## 1. المكونات المشتركة: Header

### Desktop (>1024px)
```
┌──────────────────────────────────────────────────────────────────┐
│ [شعار VERTEX]   الرئيسية  البرامج  الأكاديمية  المتجر  المدونة   │
│                                    [🔍 بحث] [🌐 AR/EN] [🛒] [👤]  │
└──────────────────────────────────────────────────────────────────┘
```
- ارتفاع 72px، خلفية بيضاء/Dark Blue حسب الوضع (Light/Dark Mode)، ظل خفيف عند التمرير (`shadow-sm` بعد `scroll > 0`).
- روابط التنقل الرئيسية بخط Medium، لون Primary عند Hover/Active.
- أيقونة السلة تعرض عدّاد (Badge) بعدد العناصر بلون Accent الذهبي.
- زر تبديل اللغة بجانب زر الحساب مباشرة.

### Tablet (640–1024px)
```
┌───────────────────────────────────────────┐
│ [☰]   [شعار VERTEX]   [🔍] [🛒] [👤]        │
└───────────────────────────────────────────┘
```
- روابط التنقل الرئيسية تُطوى داخل قائمة (☰) تنسدل كـ Drawer جانبي (shadcn `Sheet`).

### Mobile (<640px)
```
┌───────────────────────────────┐
│ [☰]     [شعار]       [🛒][👤] │
└───────────────────────────────┘
```
- البحث ينتقل لأيقونة منفصلة تفتح شريط بحث ملء العرض عند الضغط، بدل أخذ مساحة دائمة.
- التنقل الرئيسي عبر **Bottom Navigation** (منفصل عن الـ Header تمامًا) — انظر قسم Navigation.

## 2. المكونات المشتركة: Footer

### Desktop / Tablet
```
┌──────────────────────────────────────────────────────────────┐
│ [شعار + وصف مختصر]   [روابط سريعة]   [الأكاديمية]  [تواصل]    │
│                                                                │
│ [أيقونات Social Media]         النشرة البريدية [Email][اشترك] │
├──────────────────────────────────────────────────────────────┤
│ © 2026 VERTEXworkout   |  الخصوصية | الشروط | الكوكيز         │
└──────────────────────────────────────────────────────────────┘
```
- 4 أعمدة على Desktop، تتحول إلى عمودين على Tablet.

### Mobile
- كل الأعمدة تتكدّس عموديًا داخل Accordion قابل للطي (توفير مساحة)، ما عدا حقل النشرة البريدية وحقوق النشر اللذان يبقيان ظاهرين دائمًا.
- **مساحة إضافية سفلية (padding-bottom)** بمقدار ارتفاع Bottom Navigation حتى لا يتم حجب المحتوى.

## 3. المكونات المشتركة: Navigation

### Desktop/Tablet
تنقل أفقي داخل Header (موضّح أعلاه) + Breadcrumbs أسفل الـ Header مباشرة في الصفحات الداخلية (`الرئيسية > المتجر > VERTEX Power Bag`).

### Mobile — Bottom Navigation
```
┌───────────────────────────────┐
│  🏠      🏋️      📚      👤   │
│ الرئيسية  المتجر  الأكاديمية  حسابي │
└───────────────────────────────┘
```
- 4 عناصر فقط (قاعدة Thumb-Zone لعام 2026: لا يزيد عن 5 عناصر، مع تفضيل مساحة كافية لكل إبهام).
- العنصر النشط يُبرز بلون Accent الذهبي + تكبير طفيف للأيقونة (Micro-interaction).
- ثابت دائمًا (`fixed bottom-0`)، فوق كل المحتوى بما فيه الـ Footer.

## 4. صفحة: Home Page

**الهدف الأساسي:** إيصال هوية VERTEX بقوة خلال أول 3 ثوانٍ، وتوجيه الزائر لأقرب نقطة قيمة (برنامج، منتج، أو تسجيل).

### Desktop
```
┌─────────────────────────────────────────────────────────┐
│ Header                                                    │
├─────────────────────────────────────────────────────────┤
│           HERO: فيديو/صورة خلفية بطاقة كاملة العرض        │
│         [عنوان قوي] [وصف سطرين] [CTA رئيسي] [CTA ثانوي]   │
├─────────────────────────────────────────────────────────┤
│   لماذا VERTEX؟  [3-4 بطاقات مزايا بأيقونات]              │
├─────────────────────────────────────────────────────────┤
│   البرامج المميزة   [Carousel أفقي: 4 بطاقات برامج]        │
├─────────────────────────────────────────────────────────┤
│   VERTEX Academy   [شريط أقسام: Nutrition|Anatomy|...]     │
├─────────────────────────────────────────────────────────┤
│   منتجات مختارة من المتجر  [شبكة 4 منتجات]                 │
├─────────────────────────────────────────────────────────┤
│   قصص نجاح / إحصائيات  [عداد: عدد المتدربين، البرامج...]   │
├─────────────────────────────────────────────────────────┤
│   CTA ختامي: ابدأ رحلتك الآن  [زر تسجيل كبير]              │
├─────────────────────────────────────────────────────────┤
│ Footer                                                     │
└─────────────────────────────────────────────────────────┘
```
- Hero بارتفاع `100vh` أو `85vh`، مع Overlay غامق (Dark Blue شفاف) فوق الفيديو لضمان وضوح النص.
- كل قسم يفصله `padding` رأسي كبير (96px+ على Desktop) لإحساس Premium وعدم الازدحام.

### Tablet
- نفس ترتيب الأقسام، لكن البطاقات (المزايا/المنتجات) تتحول من 4 أعمدة إلى عمودين.
- Hero يقل ارتفاعه إلى `70vh`.

### Mobile
- Hero بارتفاع `60vh`، والنص يتوسّط عموديًا مع CTA بعرض كامل (Full-width button).
- كل الأقسام أعمدة مفردة، والـ Carousel يصبح قابل للسحب أفقيًا (Swipe) بمؤشرات نقطية (Dots).
- قسم "لماذا VERTEX" يتحول لبطاقات أفقية قابلة للتمرير بدل شبكة.

## 5. صفحة: Programs (قائمة البرامج)

### Desktop
```
┌─────────────────────────────────────────────────────────┐
│ Header + Breadcrumb                                       │
├───────────┬─────────────────────────────────────────────┤
│  Sidebar   │  شبكة برامج (3 أعمدة)                        │
│  فلاتر:    │  [بطاقة برنامج] [بطاقة برنامج] [بطاقة]        │
│  - الهدف   │  [بطاقة برنامج] [بطاقة برنامج] [بطاقة]        │
│  - المستوى │                                              │
│  - المدة   │  [Pagination / Load More]                    │
└───────────┴─────────────────────────────────────────────┘
```
- كل بطاقة برنامج: صورة غلاف، اسم، مستوى الصعوبة (Badge)، مدة البرنامج، السعر أو "مجاني"، زر "عرض التفاصيل".

### Tablet
- Sidebar يتحول لشريط فلاتر أفقي قابل للطي أعلى الشبكة، والشبكة تصبح عمودين.

### Mobile
- زر "فلترة" ثابت أعلى الشبكة، يفتح **Bottom Sheet** (shadcn `Sheet` من الأسفل) يحتوي كل خيارات الفلترة.
- الشبكة عمود واحد، بطاقات بعرض كامل.

## 6. صفحة: Exercise Library

### Desktop
```
┌─────────────────────────────────────────────────────────┐
│ Header + [شريط بحث كبير: ابحث عن تمرين...]                │
├───────────┬─────────────────────────────────────────────┤
│  Sidebar   │  شبكة تمارين (4 أعمدة) — كل بطاقة:            │
│  - العضلة  │  [صورة/GIF مصغّر + اسم + مستوى الصعوبة]       │
│  - المعدة  │                                              │
│  - الهدف   │                                              │
│  - الصعوبة │                                              │
└───────────┴─────────────────────────────────────────────┘
```
عند فتح تفصيل تمرين (Modal أو صفحة مستقلة حسب السياق):
```
┌─────────────────────────────────────────────────────────┐
│ [فيديو التمرين - عرض كبير]  │  [اسم التمرين]              │
│                             │  [العضلات المستهدفة - Badges]│
│                             │  [المعدات المطلوبة]          │
│                             │  [مستوى الصعوبة]             │
│                             │  [زر: إضافة لبرنامجي]        │
├─────────────────────────────────────────────────────────┤
│ Tabs: [الشرح] [الأخطاء الشائعة] [نصائح الأداء]              │
└─────────────────────────────────────────────────────────┘
```

### Tablet
- الشبكة عمودين، الفلاتر أفقية قابلة للطي (نفس نمط Programs).
- صفحة التفصيل: الفيديو أعلى، التفاصيل أسفله (عمود واحد بدل جنب لجنب).

### Mobile
- الشبكة عمود واحد ببطاقات أفقية (صورة يسار/يمين + نص) لتقليل التمرير.
- الفيديو في صفحة التفصيل يفتح بملء الشاشة تلقائيًا عند الضغط (native fullscreen video).
- الـ Tabs تتحول لـ Accordion (شرح/أخطاء/نصائح كل قسم قابل للطي) لتقليل طول الصفحة.
