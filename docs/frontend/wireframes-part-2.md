## 8. صفحة: Store (قائمة المنتجات)

### Desktop
```
┌─────────────────────────────────────────────────────────┐
│ Header + Breadcrumb                                       │
├───────────┬─────────────────────────────────────────────┤
│  Sidebar   │  [شريط ترتيب: الأحدث/الأقل سعرًا/الأعلى تقييمًا]│
│  فلاتر:    │  شبكة منتجات (3 أعمدة)                       │
│  - الفئة   │  [بطاقة منتج] [بطاقة منتج] [بطاقة منتج]       │
│  - السعر   │  [بطاقة منتج] [بطاقة منتج] [بطاقة منتج]       │
│  - التقييم │  [Pagination]                                │
└───────────┴─────────────────────────────────────────────┘
```
- بطاقة المنتج: صورة (Hover يعرض صورة ثانية)، اسم، السعر، تقييم بالنجوم، أيقونة "إضافة للمفضلة" أعلى يمين البطاقة، زر "إضافة للسلة" يظهر عند Hover.

### Tablet
- Sidebar فلاتر أفقية قابلة للطي، الشبكة عمودين.

### Mobile
- زر "فلترة وترتيب" ثابت أعلى الشبكة يفتح Bottom Sheet.
- الشبكة عمودين (وليس عمود واحد كباقي الصفحات — منتجات المتجر تحتاج كثافة بصرية أعلى لتصفح أسرع، معيار شائع في تطبيقات E-commerce لعام 2026).
- زر "إضافة للسلة" ظاهر دائمًا أسفل كل بطاقة (لا يعتمد على Hover غير المتاح باللمس).

## 9. صفحة: Product Details (تفاصيل منتج)

### Desktop
```
┌─────────────────────────────────────────────────────────┐
│ Header + Breadcrumb                                       │
├───────────────────────┬───────────────────────────────┤
│  [معرض صور - رئيسية +   │  [اسم المنتج]                   │
│   صور مصغّرة جانبية]     │  [تقييم بالنجوم + عدد المراجعات] │
│                         │  [السعر]                        │
│                         │  [اختيار المقاس/اللون - Variants]│
│                         │  [عداد الكمية] [زر أضف للسلة]    │
│                         │  [زر أضف للمفضلة]                │
│                         │  [ضمان الشحن/الإرجاع - أيقونات]  │
├─────────────────────────────────────────────────────────┤
│  Tabs: [الوصف] [المواصفات] [المراجعات]                    │
├─────────────────────────────────────────────────────────┤
│  منتجات ذات صلة (شبكة 4 منتجات)                            │
└─────────────────────────────────────────────────────────┘
```
- زر "أضف للسلة" بلون Accent الذهبي — أبرز عنصر تفاعلي في الصفحة (High-contrast CTA).

### Tablet
- المعرض والتفاصيل يبقيان جنبًا لجنب لكن بنسبة أعمدة مختلفة (50/50 بدل 60/40).

### Mobile
- معرض الصور أعلى الصفحة (Swipeable) بعرض كامل، التفاصيل أسفله بعمود واحد.
- **زر "أضف للسلة" يتحول لشريط ثابت أسفل الشاشة** (`fixed bottom`, فوق Bottom Navigation) يبقى ظاهرًا أثناء التمرير — نمط أساسي في تطبيقات E-commerce 2026 لتقليل الاحتكاك.

## 10. صفحة: Client Dashboard

### Desktop
```
┌─────────────────────────────────────────────────────────┐
│ Header                                                     │
├───────────┬─────────────────────────────────────────────┤
│  Sidebar   │  [ترحيب: أهلاً بعودتك، (الاسم)]                │
│  - الرئيسية│  [بطاقات ملخص: البرنامج الحالي | التقدم | الإنجازات]│
│  - برامجي  │                                              │
│  - التقدم  │  [البرنامج النشط - بطاقة كبيرة مع زر متابعة]   │
│  - الإنجازات│                                              │
│  - الطلبات │  [رسم بياني للتقدم - آخر 30 يوم]              │
│  - المفضلة │                                              │
│  - الإشعارات│ [آخر الأنشطة - Activity Timeline]             │
│  - الإعدادات│                                              │
└───────────┴─────────────────────────────────────────────┘
```
- Sidebar ثابت (Sticky)، العنصر النشط مُبرز بخلفية Accent فاتحة.

### Tablet
- Sidebar يتحول لأيقونات فقط (Collapsed) مع Tooltip عند Hover، قابل للتوسعة يدويًا.

### Mobile
- Sidebar يختفي تمامًا، ويُستبدل بـ Bottom Navigation العام + قائمة "المزيد" من أيقونة الحساب لباقي الخيارات (الإعدادات، المفضلة، الإشعارات).
- بطاقات الملخص تصبح Carousel أفقي قابل للسحب بدل شبكة.

## 11. صفحة: Login

### Desktop / Tablet
```
┌─────────────────────────────────────────────────────────┐
│  [صورة/فيديو رياضي - نصف الشاشة]  │  [شعار VERTEX]         │
│                                    │  [تسجيل الدخول]        │
│                                    │  [حقل البريد]          │
│                                    │  [حقل كلمة المرور]     │
│                                    │  [نسيت كلمة المرور؟]   │
│                                    │  [زر دخول - عرض كامل]  │
│                                    │  [—— أو ——]            │
│                                    │  [زر Google]           │
│                                    │  [ليس لديك حساب؟ سجل]  │
└─────────────────────────────────────────────────────────┘
```
- تقسيم 50/50: نصف بصري (يعكس الهوية القوية)، نصف وظيفي (الفورم).

### Mobile
- الصورة تختفي أو تصبح خلفية علوية قصيرة (30% من الشاشة) فوق الفورم مباشرة — الفورم يأخذ الأولوية الكاملة على الشاشات الصغيرة.

## 12. صفحة: Register

نفس تقسيم صفحة Login تمامًا (اتساق بصري)، مع فورم أطول:
```
[الاسم الكامل] [البريد الإلكتروني] [كلمة المرور] [تأكيد كلمة المرور]
[✓ أوافق على الشروط وسياسة الخصوصية]
[زر إنشاء حساب - عرض كامل]
```
- مؤشر قوة كلمة المرور (Password Strength Indicator) أسفل حقل كلمة المرور مباشرة — يظهر فور بدء الكتابة.
- على الموبايل: نفس منطق صفحة Login (الفورم أولاً).

## 13. صفحة: Cart (سلة التسوق)

### Desktop
```
┌─────────────────────────────────────────────────────────┐
│ Header + Breadcrumb                                       │
├───────────────────────────────────┬─────────────────────┤
│  [قائمة عناصر السلة]                │  [ملخص الطلب]         │
│  [صورة][اسم][مقاس][سعر][كمية][حذف]  │  المجموع الفرعي       │
│  [صورة][اسم][مقاس][سعر][كمية][حذف]  │  الشحن                │
│                                     │  الإجمالي             │
│  [كود خصم - حقل + زر تطبيق]          │  [زر متابعة الدفع]    │
└───────────────────────────────────┴─────────────────────┘
```

### Tablet
- نفس التقسيم لكن بنسبة أعمدة أضيق للملخص.

### Mobile
- عناصر السلة عمود واحد (كل عنصر بطاقة كاملة العرض).
- ملخص الطلب ينتقل لأسفل قائمة العناصر، مع **زر "متابعة الدفع" ثابت أسفل الشاشة**.

## 14. صفحة: Checkout

### Desktop
```
┌─────────────────────────────────────────────────────────┐
│ Header (مبسّط - بدون تنقل رئيسي، فقط الشعار + خطوات)        │
├─────────────────────────────────────────────────────────┤
│  [مؤشر خطوات: 1.الشحن  2.الدفع  3.التأكيد]                 │
├───────────────────────────────────┬─────────────────────┤
│  [فورم العنوان / اختيار عنوان محفوظ]│  [ملخص الطلب المصغّر]  │
│  [خيارات بوابة الدفع: Paymob/Stripe]│  (نفس تفاصيل السلة)   │
│  [زر تأكيد الطلب]                   │                     │
└───────────────────────────────────┴─────────────────────┘
```
- **Header مبسّط عمدًا** (بدون روابط تنقل تشتت الانتباه) — مبدأ "Distraction-Free Checkout" الأساسي لتقليل التخلي عن السلة (Cart Abandonment).

### Tablet
- نفس المنطق، الملخص ينتقل أسفل الفورم بدل جانبه إذا ضاقت المساحة.

### Mobile
- مؤشر الخطوات يتحول لشريط تقدم بسيط (Progress Bar) بدل نص الخطوات الثلاث.
- كل خطوة تُعرض في شاشة منفصلة (Wizard Pattern) بدل صفحة طويلة واحدة — تقليل الإحساس بالتعقيد على الشاشة الصغيرة.
- ملخص الطلب يظهر كـ Accordion قابل للطي أعلى الفورم ("عرض ملخص الطلب ▾") لتوفير مساحة.
