## 15. صفحة: Global Search — `/[locale]/search`

### Desktop
```
┌─────────────────────────────────────────────────────────┐
│ Header                                                     │
├─────────────────────────────────────────────────────────┤
│  [شريط بحث كبير مركزي: "ابحث عن منتج، برنامج، تمرين..."]   │
├───────────┬─────────────────────────────────────────────┤
│  Sidebar   │  Tabs: [الكل][منتجات][برامج][تمارين][أكاديمية][مدونة]│
│  فلاتر:    │                                              │
│  - النوع   │  نتائج موحّدة (شبكة/قائمة حسب النوع)           │
│  - السعر   │  كل نتيجة: [صورة مصغّرة][عنوان][نوع - Badge][ملخص]│
│  - الفئة   │                                              │
│  ترتيب:    │  [Pagination]                                │
│  - الصلة   │                                              │
│  - الأحدث  │                                              │
└───────────┴─────────────────────────────────────────────┘
```
- تبويب "الكل" يعرض أفضل 3 نتائج من كل نوع مع رابط "عرض كل نتائج [النوع]".
- كل نتيجة تحمل Badge يوضح نوعها (منتج/برنامج/تمرين/مقال) بلون مختلف لتمييز سريع.

### Tablet
- الفلاتر تتحول لشريط أفقي قابل للطي أعلى النتائج، Tabs تبقى أفقية Scrollable.

### Mobile
- شريط البحث ثابت أعلى الشاشة (Sticky) أثناء التمرير.
- Tabs الأنواع Scrollable أفقيًا.
- زر "فلترة وترتيب" يفتح Bottom Sheet (نفس نمط باقي الصفحات).

## 16. صفحة: Profile Edit — `/[locale]/profile/edit`

مستقلة عن `dashboard/profile` (العرض) — هذه مخصصة للتعديل الفعلي فقط، ويُوصل إليها أيضًا بزر "تعديل الملف الشخصي" من داخل Dashboard.

### كل الأجهزة (نفس الهيكل، عمود واحد يتسع/يضيق)
```
┌─────────────────────────────────────────┐
│  [صورة شخصية + زر تغيير الصورة]           │
├─────────────────────────────────────────┤
│  Tabs: [البيانات الأساسية][القياسات][كلمة المرور][الإشعارات]│
├─────────────────────────────────────────┤
│  [فورم القسم النشط]                       │
│  [زر حفظ التغييرات] [زر إلغاء]             │
└─────────────────────────────────────────┘
```
- Desktop: Tabs أفقية أعلى الفورم. Mobile: Tabs تتحول لقائمة منسدلة (Select) لتوفير مساحة.
- حفظ تلقائي للمسودة (Auto-save draft) اختياري لاحقًا — غير مفعّل في Phase 1.

## 18. توثيق شامل لكل Empty States

| الصفحة | الرسالة | السبب المحتمل المعروض | CTA |
|---|---|---|---|
| **لا توجد برامج** (نتيجة فلترة) | "لم نجد برامج مطابقة" | "جرّب تعديل الفلاتر أو إزالتها" | زر "إزالة كل الفلاتر" |
| **لا توجد منتجات** (تصنيف فارغ) | "لا توجد منتجات في هذا التصنيف حاليًا" | "قد تكون نفدت من المخزون مؤقتًا" | زر "تصفّح كل المنتجات" |
| **لا توجد طلبات** | "لم تقم بأي طلب بعد" | — | زر "ابدأ التسوق الآن" → المتجر |
| **لا توجد إشعارات** | "لا توجد إشعارات جديدة" | "سنخبرك فور وجود تحديث يخصك" | بدون CTA (حالة طبيعية غير سلبية) |
| **نتائج بحث فارغة** | "لم نجد نتائج لـ '[كلمة البحث]'" | "تحقق من الإملاء أو جرّب كلمات أعم" | اقتراح 3 كلمات بحث شائعة |
| **المفضلة فارغة** | "لم تُضف أي عنصر للمفضلة بعد" | — | زر "اكتشف المنتجات" → المتجر |
| **السلة فارغة** | "سلتك فارغة حاليًا" | — | زر "تصفّح المتجر" (بارز، بلون Accent) |

**مبدأ التصميم لكل Empty State:** أيقونة/رسم توضيحي بسيط بخط Line Art (متوافق مع الهوية، ليس كرتونيًا) + العنوان + السبب (إن وجد) + CTA واحد واضح فقط — لا خيارات متعددة تشتت المستخدم.

## 19. معايير الأداء أثناء التحميل (عبر كل الصفحات)

| التقنية | أين تُطبَّق | التفصيل |
|---|---|---|
| **Skeleton Loading** | كل قائمة/شبكة بيانات (منتجات، برامج، تمارين، طلبات) | شكل هيكلي يحاكي البطاقة الفعلية (نفس الأبعاد)، لا Spinner عام |
| **Lazy Loading** | كل صورة/فيديو أسفل "طية الصفحة" (Below the Fold) | تحميل عند الاقتراب من منطقة الرؤية فقط (`loading="lazy"` / Next.js Image تلقائيًا) |
| **Progressive Image Loading** | صور المنتجات، التمارين، الأغلفة | نسخة مصغّرة ضبابية (Blur Placeholder) تظهر فورًا ثم تُستبدل بالصورة كاملة الدقة |
| **Image Placeholder** | أي صورة قبل التحميل الكامل | لون خلفية محايد (`neutral-100`) بنفس أبعاد الصورة النهائية لمنع Layout Shift |
| **Infinite Scroll** | مكتبة التمارين، المدونة، نتائج البحث (محتوى استكشافي) | تحميل تلقائي عند الاقتراب من نهاية الصفحة |
| **Pagination** | المتجر، البرامج، الطلبات، لوحة الأدمن (محتوى يحتاج "رجوع لصفحة محددة") | أرقام صفحات واضحة — أنسب من Infinite Scroll عند الحاجة للوصول لعنصر محدد لاحقًا |

**القاعدة الفاصلة بين Infinite Scroll وPagination:** المحتوى الاستكشافي (تصفح حر بدون هدف محدد) → Infinite Scroll. المحتوى الإداري/التاريخي (يحتاج المستخدم الرجوع لصفحة بعينها) → Pagination.
