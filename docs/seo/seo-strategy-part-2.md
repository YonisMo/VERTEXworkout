## 9. Internal Linking Strategy

**الهدف:** توزيع "قوة" الصفحات (Link Equity) بذكاء، ومساعدة Google على فهم العلاقة بين الأقسام المختلفة (متجر ↔ أكاديمية ↔ مكتبة تمارين ↔ برامج).

| نمط الربط | مثال فعلي |
|---|---|
| **ربط سياقي داخل المحتوى** | مقال أكاديمية عن "التغذية لبناء العضلات" يربط داخل النص لمنتج "VERTEX Power Bag" ولبرنامج "تضخيم 12 أسبوع" ذوي الصلة الفعلية |
| **قسم "محتوى ذو صلة" الموحّد** | مبني فعليًا على نظام `tags`/`taggables` العام (Database Schema، القسم 6) — تمرين "ضغط بار" مرتبط تلقائيًا بمقال "تشريح عضلات الصدر" عبر Tag مشترك `chest` |
| **Breadcrumbs** | رابط صاعد دائم لكل مستوى أعلى (القسم 8) |
| **الروابط من الصفحة الرئيسية** | تُحدَّث دوريًا لتُبرِز أحدث/أهم المحتوى (برامج مميزة، مقالات جديدة) — تضخ "قوة" من أقوى صفحة في الموقع (الرئيسية) لأحدث المحتوى المُراد ترتيبه سريعًا |

**قاعدة صارمة:** كل صفحة محتوى (منتج، برنامج، مقال، تمرين) يجب أن تكون قابلة للوصول بحد أقصى **3 نقرات من الرئيسية** (متوافق مع Sitemap، القسم 1) — عمق أكبر يقلل احتمال زحف Google لها بانتظام.

## 10. Image SEO

| الممارسة | التفصيل |
|---|---|
| **`alt` text وصفي دائمًا** | من حقل `media.alt_text` (Database Schema) — **إلزامي** لكل صورة، ليس فقط لإمكانية الوصول بل يُقرَأ من Google Image Search أيضًا؛ يصف المحتوى الفعلي: `"VERTEX Power Bag أسود 15 كيلوجرام - كيس تدريب وزني"` وليس `"صورة1"` أو اسم الملف الخام |
| **اسم ملف وصفي قبل الرفع** (على مستوى تخزين `media`) | `vertex-power-bag-15kg-black.webp` بدل `IMG_2847.webp` — يُشتق تلقائيًا من slug المنتج عند الرفع |
| **Sitemap صور مخصص** | `sitemap-images.xml` يضيف كل صور المنتجات لفهرسة أسرع في Google Images — قناة اكتشاف إضافية مهمة لمتجر إلكتروني |
| **الأبعاد والصيغة** | صيغ حديثة (WebP/AVIF) وأبعاد محسَّنة (موضّحة في Performance Strategy) — سرعة تحميل الصورة نفسها عامل تصنيف SEO مباشر |

## 11. Video SEO

لفيديوهات مكتبة التمارين تحديدًا:
- **`VideoObject` Schema** لكل صفحة تمرين (اسم، وصف، مدة، صورة مصغّرة، رابط الفيديو) — يسمح لـ Google بعرض الفيديو مباشرة في نتائج البحث مع Rich Snippet.
- **Sitemap فيديو مخصص** (`sitemap-videos.xml`) لتسريع اكتشاف Google لمحتوى الفيديو الجديد.
- **الترجمة النصية (Transcript)** لكل فيديو تمرين تُضاف كنص مصاحب (من حقل `description`/`performance_tips` الموجود أصلاً في Database Schema) — يمنح Google محتوى نصيًا كاملاً يفهرسه حتى لو لم "يشاهد" الفيديو فعليًا.

## 12. Content SEO

**استراتيجية المحتوى (VERTEX Academy + Blog) كأداة SEO أساسية، وليست ثانوية:**

| المبدأ | التطبيق |
|---|---|
| **بناء سلطة الموضوع (Topical Authority)** | تغطية شاملة لكل موضوع فرعي ضمن كل قسم أكاديمي (Nutrition, Anatomy, Mobility...) — وليس مقالات متفرقة عشوائية؛ يُبنى تقويم محتوى (Content Calendar) يغطي "عناقيد مواضيع" (Topic Clusters) كاملة |
| **مقالة "ركيزة" لكل قسم (Pillar Content)** | مقال شامل طويل لكل قسم رئيسي (مثال: "الدليل الشامل للتغذية الرياضية") يربط داخليًا لكل المقالات الفرعية الأكثر تخصصًا ضمن نفس الموضوع |
| **تحديث المحتوى القديم دوريًا** | مقالات الأكاديمية تُراجَع كل 6 أشهر للتأكد من دقتها وتحديث `updated_at` فعليًا — Google يفضّل محتوى محدَّثًا فعليًا (وليس فقط تغيير التاريخ شكليًا دون تعديل حقيقي) |
| **طول المحتوى المناسب** | لا حد أدنى تعسفي بالكلمات — الطول يخدم شمولية الإجابة على نية البحث فعليًا، وليس حشوًا لتحقيق رقم معين |

## 13. Multi-language SEO (تعميق كامل)

**هذا القسم حرج جدًا لأن معظم أخطاء SEO الثنائية اللغة قاتلة إن لم تُنفَّذ بدقة:**

```html
<!-- في كل صفحة، الإشارة الكاملة لكل النسخ اللغوية المتاحة -->
<link rel="alternate" hreflang="ar" href="https://vertexworkout.com/ar/store/product/vertex-power-bag" />
<link rel="alternate" hreflang="en" href="https://vertexworkout.com/en/store/product/vertex-power-bag" />
<link rel="alternate" hreflang="x-default" href="https://vertexworkout.com/en/store/product/vertex-power-bag" />
```
| القاعدة | لماذا |
|---|---|
| **hreflang في كل صفحة بدون استثناء** | يمنع Google من معاملة النسخة العربية والإنجليزية كمحتوى مكرر (Duplicate Content) عبر نطاقين لغويين مختلفين لنفس الرابط الأساسي |
| **`x-default` يُشير للإنجليزية** | النسخة الافتراضية لزوار من دول لا تُحدَّد لغتهم تلقائيًا بدقة |
| **الترابط الكامل (Reciprocal)** | كل صفحة عربية تُشير للإنجليزية **والعكس** — إشارة أحادية الاتجاه فقط تُبطل الفائدة كليًا في نظر Google |
| **Slug مترجَم أم موحَّد؟** | **القرار المعتمد: الـ Slug يبقى كما هو (لاتيني) في كلا اللغتين** (`vertex-power-bag` وليس `كيس-قوة-فيرتكس`) — يبسّط الصيانة ويتفادى مشاكل الترميز في الروابط، بينما العنوان الفعلي (`<title>`, `<h1>`) هو ما يُترجَم فعليًا ويُقرَأ من محركات البحث كنص |
| **Sitemap منفصل لكل لغة أم موحَّد؟** | موحَّد (كما في القسم 5) — يحتوي روابط كلا اللغتين معًا، وGoogle يربط بينها تلقائيًا عبر إشارات `hreflang` الموجودة في كل صفحة |

## 14. Local SEO

رغم أن VERTEXworkout منصة رقمية بالأساس (لا صالة فعلية حاليًا)، الأولوية الجغرافية (مصر/المنطقة العربية) تُدار عبر:
- **Google Business Profile** (إن وُجد مقر فعلي مستقبلاً) — غير مطلوب في MVP.
- **استهداف لغوي/جغرافي عبر المحتوى نفسه:** مقالات ونماذج تسعير بالجنيه المصري (EGP) كعملة أساسية، أمثلة ومراجع ثقافية مناسبة للجمهور المصري/العربي.
- **`hreflang` + استضافة على Edge قريب جغرافيًا** (موضّح في Performance Strategy، القسم 6) يخدمان معًا إشارة "صلة جغرافية" غير مباشرة لمحركات البحث.

## 15. Core Web Vitals & SEO

**العلاقة المباشرة:** Core Web Vitals (Performance Strategy، القسم 20) هي **عامل تصنيف رسمي مُعلَن من Google** (Page Experience Signals) منذ 2021 — ليست "مجرد" مقياس أداء تقني منفصل عن SEO، بل جزء لا يتجزأ منه. **لا حاجة لتكرار التفاصيل هنا** — الالتزام الكامل بأهداف Performance Strategy (القسم 20) **هو نفسه** الالتزام بهذا الجانب من SEO.

## 16. Indexing Strategy

| القرار | التفصيل |
|---|---|
| **صفحات تُفهرَس صراحة** | كل الصفحات العامة (منتجات، برامج، تمارين، محتوى) — `robots: { index: true, follow: true }` |
| **صفحات تُمنَع من الفهرسة صراحة** | `/dashboard/*`, `/cart`, `/checkout/*`, `/login`, `/register` — لا قيمة بحثية، وقد تُربك النتائج (`robots: { index: false }` + استبعاد من `robots.txt`) |
| **صفحات نتائج فلترة مركّبة معقّدة** (`?category=X&minPrice=Y&color=Z` معًا) | `noindex, follow` — لا تُفهرَس بذاتها (منع تضخم عدد الصفحات منخفضة القيمة - Index Bloat) لكن تُتبَع روابطها الداخلية لصفحات المنتج الفعلية |
| **صفحة البحث (`/search`)** | `noindex, follow` بنفس المنطق — نتائج بحث ديناميكية لا قيمة لها كصفحة مفهرَسة مستقلة |

## 17. Crawl Budget Optimization

مهم خصوصًا مع نمو كتالوج المنتجات والمحتوى:
- **منع زحف الروابط منخفضة القيمة عبر `robots.txt`** (فلاتر مركّبة، صفحات إدارية) — يوجّه "ميزانية الزحف" المحدودة لـ Google نحو الصفحات ذات القيمة الفعلية بدل إهدارها على آلاف تركيبات الفلاتر الممكنة.
- **تقليل الروابط اليتيمة (Orphan Pages)** — كل صفحة منتج/مقال يجب أن تكون مرتبطة من صفحة تصنيف/قائمة واحدة على الأقل (يُفحص هذا آليًا عبر سكربت يقارن كل الـ Slugs في قاعدة البيانات مقابل الروابط الداخلية الفعلية المكتشفة).
- **تحديث `lastmod` بدقة** (القسم 5) — يساعد Google على تحديد أولويات إعادة الزحف للمحتوى المُحدَّث فعليًا بدل إهدار الميزانية على صفحات ثابتة لم تتغيّر.

## 18. Redirect Strategy

| الحالة | نوع التحويل |
|---|---|
| تغيير Slug منتج/مقال (نادر، لكن يحدث) | `301 Permanent Redirect` من الرابط القديم للجديد — **إلزامي**، لا يُترك الرابط القديم يُرجع 404 فقط |
| حذف منتج نهائيًا (نفدت الكمية للأبد، توقف الإنتاج) | `301` لصفحة التصنيف الأقرب (وليس للرئيسية العامة) — يحافظ على قيمة SEO للرابط القديم بأقرب صلة ممكنة |
| صفحات مؤقتة (صيانة) | `307 Temporary Redirect` — لا يُفقد الفهرسة الأصلية لأن التحويل معلوم أنه مؤقت |
| **قاعدة صارمة** | **لا سلاسل تحويل (Redirect Chains)** — أي `A → B → C` يُصحَّح فورًا لـ `A → C` مباشرة؛ يُفحص هذا دوريًا عبر أداة زحف (Screaming Frog أو مكافئ) |

جدول `redirects` مركزي في قاعدة البيانات (أو `next.config.js` للتحويلات الثابتة القليلة) يُدار من لوحة تحكم الأدمن مستقبلاً (Phase 3) لتسهيل إضافة تحويلات دون نشر كود جديد.

## 19. SEO Monitoring & Search Console Integration

| الأداة | الاستخدام |
|---|---|
| **Google Search Console** | مربوطة من اليوم الأول (حتى قبل الإطلاق العام)، Sitemap مُرسَل يدويًا أول مرة، مراقبة أسبوعية لـ: أخطاء الزحف، تغطية الفهرسة، الكلمات المفتاحية الفعلية التي يصل الموقع عبرها |
| **Bing Webmaster Tools** | مربوطة أيضًا (حصة سوقية غير مهملة، إعداد سريع بعد Google) |
| **تنبيهات تلقائية** | أي انخفاض مفاجئ في عدد الصفحات المفهرَسة (Search Console API) → تنبيه Slack فوري للفريق (قد يشير لخطأ تقني حرج مثل `robots.txt` معطوب بالخطأ) |
| **تتبع الترتيب (Rank Tracking)** | أداة خارجية (مثل Ahrefs/SEMrush) لتتبع ترتيب أهم 20-30 كلمة مفتاحية مستهدفة أسبوعيًا |

## 20. Rich Results Optimization

بناءً على Structured Data (القسم 6)، التحقق الفعلي من ظهور Rich Snippets:
- **اختبار كل نوع Schema عبر Google Rich Results Test** قبل الإطلاق وبعد أي تعديل جوهري على القوالب.
- **مراقبة تقرير "Enhancements" في Search Console** (Products, Courses, Breadcrumbs) — يوضح أخطاء أو تحذيرات فعلية في البيانات المنظَّمة المنشورة.
- **أولوية Rich Results للمنتجات** (نجوم التقييم + السعر في نتيجة البحث مباشرة) — الأكثر تأثيرًا على معدل النقر (CTR) لمتجر إلكتروني تحديدًا.

## ✅ SEO Production Checklist

- [ ] `hreflang` + `x-default` موجودان في **كل** صفحة بدون استثناء، ترابط ثنائي الاتجاه مؤكَّد.
- [ ] Sitemap Index مُقسَّم (static/products/programs/content/images/videos) ومُرسَل لـ Search Console وBing.
- [ ] `robots.txt` يمنع الزحف للمسارات الصحيحة فقط (لا حجب خاطئ لصفحات يجب فهرستها).
- [ ] كل صفحة عامة تحمل `<title>` و`description` فريدين (تم فحصه آليًا، صفر تكرار).
- [ ] Structured Data لكل الأنواع السبعة (القسم 6) تم اختبارها عبر Rich Results Test بدون أخطاء.
- [ ] Canonical صحيح لكل صفحة (خصوصًا صفحات الفلاتر → تُشير للنسخة الأساسية).
- [ ] كل صورة تحمل `alt` text وصفيًا حقيقيًا (لا نص عام مكرر).
- [ ] Core Web Vitals ضمن الأهداف على بيانات حقيقية (مرجع Performance Strategy).
- [ ] لا سلاسل Redirects، ولا روابط داخلية معطوبة (404 داخلية) — تم الفحص عبر أداة زحف كاملة للموقع.
- [ ] Google Search Console + Bing Webmaster Tools مربوطتان ومُفعَّلتان قبل الإطلاق العام.
- [ ] تقويم محتوى (Content Calendar) لـ VERTEX Academy مُعتمَد لأول 3 أشهر بعد الإطلاق.
